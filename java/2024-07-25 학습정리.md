# 추상화
```
여러 가지 사물이나 개념에서 공통되는 틍성이나 속성을 추출하는것.
이때 공통되는것 이외에 것들은 배제하는 사상을 수반
이미 완성된 형태의 메소드를 만들 수 있고 필요한 부분만 만들 수 있다. : 불완전추상화
공통된 것들을 하나의 추상클래스에서 관리
즉, 같은것들은 가져오고 다른것들은 버리자!!!
```
* 추상클래스
```
구현하지 않은 메소드 == Abstract Method
class 이름 앞에 Abstract를 붙인다.
```
## 인터페이스
```
정해진 규격에 의해서 통신되는것
서로 다른 주체 사이의 통신을 위한 규약
서로 다른 두 시스템, 장치,소프트 웨어 따위를 서로 이어 주는 부분
오로지 추상메소드 abstract만 사용하기때문에 생략 가능하다.
메소드를 완성시킬 수 없다.
100퍼센트 완전 추상화를 쓸때 사용한다.
```
* Java 인터페이스
```
클래스와 클래스가 상호작용 할 수 있도록 표준 제공
개발자 간 커뮤니케이션을 위한 표준 제공
```
* 인터페이스의 다형성
```
하나의 인터페이스를 통해 여러 개의 클래스를 생성할 수 있다.
IS A 관계
ex) TV와 리모콘이 통신하는 표준 규격을 통해 여러 형태의 리모콘이 TV와 연결될 수 있다.
```
